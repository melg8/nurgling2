# Qwen Agent Workflow: Producer → Compilation → Testing

## Описание процесса

Этот документ описывает рабочий процесс для разделения задач между суб-агентами с циклом обратной связи.

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│                        РОДИТЕЛЬСКИЙ АГЕНТ                        │
│  (координирует процесс, проверяет выполнение задачи)            │
└─────────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌─────────────────┐    ┌───────────────┐
│   PRODUCER    │    │   COMPILATION   │    │    TESTING    │
│   (код)       │───▶│   (сборка)      │───▶│   (тесты)     │
└───────────────┘    └─────────────────┘    └───────────────┘
        ▲                     │                     │
        │                     ▼                     │
        │              [ОШИБКА?]                    │
        │                     │                     │
        └─────────────────────┴─────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │  Возврат ошибки │
                    │  на доработку   │
                    └─────────────────┘
```

## Роли агентов

### 1. Producer Agent (Генератор кода)

**Задача:** Создание и модификация кода

**Входные данные:**
- Описание задачи от родителя
- Список ошибок компиляции (если есть)
- Список проваленных тестов (если есть)

**Выходные данные:**
- Изменённые файлы кода
- Описание внесённых изменений

**Промпт для запуска:**
```
Ты Producer Agent. Твоя задача:
1. Изучи задачу: {TASK_DESCRIPTION}
2. {Если есть ошибки: "Исправь следующие ошибки компиляции: {ERRORS}"}
3. {Если есть ошибки тестов: "Исправь проваленные тесты: {TEST_ERRORS}"}
4. Внеси необходимые изменения в код
5. Предоставь список изменённых файлов и описание изменений
```

---

### 2. Compilation Agent (Сборка)

**Задача:** Компиляция проекта и обнаружение ошибок

**Входные данные:**
- Путь к проекту
- Команда сборки

**Выходные данные:**
- ✅ Успех: "Сборка успешна"
- ❌ Ошибка: Список ошибок компиляции

**Промпт для запуска:**
```
Ты Compilation Agent. Твоя задача:
1. Запусти сборку проекта командой: {BUILD_COMMAND}
2. Если сборка успешна — верни: "✅ Сборка успешна"
3. Если есть ошибки — верни полный текст ошибок для передачи Producer
```

**Команды сборки для проекта:**
- `ant bin` — основная сборка
- `ant clean && ant bin` — чистая сборка
- `ant jar` — сборка JAR

---

### 3. Testing Agent (Тестирование)

**Задача:** Запуск тестов и обнаружение провалов

**Входные данные:**
- Путь к проекту
- Команда запуска тестов

**Выходные данные:**
- ✅ Успех: "Все тесты пройдены"
- ❌ Ошибка: Список проваленных тестов с ошибками

**Промпт для запуска:**
```
Ты Testing Agent. Твоя задача:
1. Запусти тесты командой: {TEST_COMMAND}
2. Если все тесты пройдены — верни: "✅ Все тесты пройдены"
3. Если есть провалы — верни названия тестов и текст ошибок для передачи Producer
```

**Команды тестирования для проекта:**
- `ant test` — запуск тестов
- `mvn test` — если используется Maven
- `./gradlew test` — если используется Gradle

---

### 4. Parent Agent (Родительский координатор)

**Задача:** Координация процесса и проверка полноты выполнения

**Алгоритм работы:**

```
1. Получить задачу от пользователя
2. Запустить Producer → получить код
3. Запустить Compilation → проверить сборку
   ├─ ❌ Ошибка → вернуть ошибки Producer → повторить с шага 2
   └─ ✅ Успех → продолжить
4. Запустить Testing → проверить тесты
   ├─ ❌ Ошибка → вернуть ошибки тестов Producer → повторить с шага 2
   └─ ✅ Успех → продолжить
5. Проверить полноту выполнения задачи
   ├─ ❌ Выполнена частично → обновить задачу → повторить с шага 2
   └─ ✅ Выполнена полностью → завершить
```

**Чек-лист проверки полноты:**
- [ ] Все требования задачи выполнены
- [ ] Нет временных решений или заглушек
- [ ] Код соответствует стилю проекта
- [ ] Добавлены тесты для нового функционала
- [ ] Обновлена документация (если требуется)

## Шаблон для запуска рабочего процесса

### Шаг 1: Запуск Producer

```markdown
## Задача для Producer Agent

**Описание задачи:** {описание}

**Требования:**
- {требование 1}
- {требование 2}

**Контекст проекта:**
- Язык: Java 21
- Сборка: Apache Ant
- Тесты: JUnit

Внеси необходимые изменения в код и предоставь список файлов.
```

### Шаг 2: Запуск Compilation

```markdown
## Задача для Compilation Agent

**Проект:** E:\work\nurgling2

**Команда сборки:** ant bin

**Действия:**
1. Запусти сборку
2. Если есть ошибки — предоставь полный текст для передачи Producer
3. Если успешно — подтверди
```

### Шаг 3: Запуск Testing

```markdown
## Задача для Testing Agent

**Проект:** E:\work\nurgling2

**Команда тестов:** ant test

**Действия:**
1. Запусти тесты
2. Если есть провалы — предоставь названия тестов и ошибки
3. Если успешно — подтверди
```

### Шаг 4: Проверка полноты (Parent)

```markdown
## Проверка полноты выполнения задачи

**Исходная задача:** {описание}

**Что было сделано:**
{список изменений от Producer}

**Чек-лист:**
- [ ] Все требования выполнены
- [ ] Нет заглушек и TODO
- [ ] Код следует стилю проекта
- [ ] Тесты добавлены/обновлены
- [ ] Документация обновлена

**Вердикт:** ✅ Полностью / ⚠️ Частично

{Если частично: описание недостающих частей для нового цикла}
```

## Примечания

1. **Ограничение глубины:** Суб-агенты не могут создавать дочерние суб-агенты с передачей данных. Все агенты запускаются от имени родителя.

2. **Передача контекста:** Каждый агент получает полный контекст от родителя, включая историю предыдущих циклов.

3. **Максимум циклов:** Рекомендуется ограничить количество циклов итерации (например, 5) для предотвращения бесконечных повторений.

4. **Логирование:** Сохраняй историю всех циклов для отладки и анализа.
