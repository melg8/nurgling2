## Why

При переходе через порталы (cave, minehole, ladder) между слоями карты игроки не видят связи между маркерами на разных уровнях. Это создаёт путаницу: одинаковые маркеры на поверхности и под землёй не выглядят как связанные переходы. Автоматическое связывание маркеров с IN/OUT суффиксами и уникальным идентификатором решит эту проблему и улучшит навигацию между слоями.

## What Changes

- **Автоматическое создание связанных маркеров** при переходе игрока через портал между слоями карты
- **Уникальный 6-символьный UID** (0-9a-zA-Z) для идентификации пары маркеров
- **Суффиксы IN/OUT** для указания направления движения (вход/выход из портала)
- **Поддержка всех типов порталов**: Cave (поверхность ↔ подземелье), Minehole/Ladder (между уровнями подземелья)
- **Дедупликация маркеров** - предотвращение создания дубликатов при повторном проходе через тот же портал
- **Логирование событий** для отладки (переходы между слоями, создание маркеров)

## Capabilities

### New Capabilities
- `portal-marker-linking`: Автоматическое связывание маркеров порталов между слоями карты. Включает обнаружение переходов между слоями, генерацию UID, создание парных маркеров IN/OUT, дедупликацию и логирование событий.

## Impact

- **Новые файлы**: `PortalMarkerLinker.java`, `PortalMarkerTracker.java`, `LayerTransition.java`, `MarkerUidGenerator.java`, `PortalMarkerLink.java`, `PortalName.java`, `PortalMarkerLogger.java`
- **Модификация**: Нет (система полностью независима от существующих компонентов)
- **Зависимости**: MapFile API для создания SMarker (системных маркеров), MCache для получения gridId
- **Конфигурация**: Новый ключ `portalMarkerAutoCreate` для включения/выключения функции
- **Логирование**: Новые лог-файлы в `logs/portal_transitions.log`, `logs/marker_events.log`, `logs/uid_generation.log`
